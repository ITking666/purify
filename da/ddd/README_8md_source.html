7 Description
8 -------------
9 
10 **PURIFY** is an open-source collection of routines written in `C++` available under the [license](#license) below. It implements different tools and high-level to perform radio interferometric imaging, _i.e._ to recover images from the Fourier measurements taken by radio interferometric telescopes.
11 
12 **PURIFY** leverages recent developments in the field of compressive sensing and convex optimization. Low-level functionality to solve the resulting convex optimisation is factored into the open-source companion code, [**SOPT**](https://github.com/astro-informatics/sopt), also written by the authors of **PURIFY**. For further background please see the [reference](#references-and-citation) section.
13 
14 This documentation outlines the necessary and optional [dependencies](#dependencies-installation) upon which **PURIFY** should be built, before describing [installation](#installing-and-building-PURIFY), [testing](#testing) and [usage](#usage) details. [Contributors](#contributors), [references](#references-and-citation) and [license](#license) information then follows.

16 Dependencies installation
17 -------------------------
18 
19 **PURIFY** is written in `C++11`. Pre-requisites and dependencies are listed in following and minimal versions required are tested against `Travis CI` meaning that they come natively with OSX and the Ubuntu Trusty release. These are also the default ones fetched by `CMake` (an internet connection is required for this).
20 
21 `C++` minimal dependencies:
22 
23 - [CMake](http://www.cmake.org/) v3.9.2 A free software that allows cross-platform compilation
24 - [GCC](https://gcc.gnu.org) v7.3.0 GNU compiler for `C++`
25 - [UCL/GreatCMakeCookOff](https://github.com/UCL/GreatCMakeCookOff) Collection of `CMake` recipes.
26  Downloaded automatically if absent
27 - [fftw3](www.fftw.org): Fastest Fourier Transform in the West
28 - [Eigen3](http://eigen.tuxfamily.org/index.php?title=Main_Page) v3.2.0 (Trusty) Modern `C++` linear algebra.
29  Downloaded automatically if absent.
30 - [tiff](http://www.libtiff.org/) v4.0.3 (Trusty) Tag Image File Format library
31 - [astro-informatics/sopt](https://github.com/astro-informatics/sopt) v* Sparse Optimization
32  Compressed Sensing library. Downloaded automatically if absent.
33 - [cfitsio](http://heasarc.gsfc.nasa.gov/fitsio/fitsio.html) v* Library of `C` and `Fortran` subroutines
34  for reading and writing data files in FITS (Flexible Image Transport System) data format.
35  Downloaded automatically if absent.
36 - [casacore](http://casacore.github.io/casacore/) - Optional - Needed to interface with measurement
37  sets. The main purify program requires this library (and its dependencies)
38 - [OpenMP](http://openmp.org/wp/) v4.8.4 (Trusty) - Optional - Speeds up some of the operations.
39 - [MPI](https://www.open-mpi.org) v3.1.1 (Trusty) - Optional - Parallelisation paradigm to speed up operations.
40 - [spdlog](https://github.com/gabime/spdlog) v* - Optional - Logging library. Downloaded automatically if
41  absent.
42 - [Catch2](https://github.com/catchorg/Catch2) v2.3.0 - Optional - A `C++`
43  unit-testing framework only needed for testing. Downloaded automatically if absent.
44 - [google/benchmark](https://github.com/google/benchmark) `v1.3.0` - Optional - A `C++`
45  micro-benchmarking framework only needed for benchmarks. Downloaded automatically if absent.

47 Installing and building PURIFY
48 -------------------------------------
49 
50 **PURIFY** can be installed through the software packet manager on Linux Debian distributions:
51 
52 ```
53 apt-get install purify
54 ```
55 
56 Alternatively, you can build **PURIFY** entirely from the source code. Once the mandatory dependencies are present, `git clone` from the [GitHub repository](https://github.com/astro-informatics/purify):
57 
58 ```
59 git clone https://github.com/astro-informatics/purify.git
60 ```
61 
62 Then, the program can be built with standard `CMake` command:
63 
64 ```
65 cd /path/to/code
66 mkdir build
67 cd build
68 cmake -DCMAKE_BUILD_TYPE=Release ..
69 make
70 ```
71 
72 To test everything went all right:
73 
74 ```
75 cd /path/to/code/build
76 ctest .
77 ```
78 
79 To install in directory `/X`, with libraries going to `X/' do:
80 
81 ```
82 cd /path/to/code/build
83 cmake -DCMAKE_INSTALL_PREFIX=/X ..
84 make install
85 ```

97 Usage
98 ------
99 
100 The main purify executable lives either in the build directory or in the in the `bin` subdirectory
101 of the installation directory. `purify` has one required argument, it a string for the file path of the config file containg the settings.
102 
103 `purify path/to/config.yaml`.
104 
105 A template with a description of the settings can be found at https://github.com/astro-informatics/purify/blob/master/data/config/config.yaml.
106 This can also be found in the downloaded repo at `data/config/config.yaml`. When `purify` runs a directory will be created, and the output images will be saved and time-stamped. Additionally, a config file with the settings used will be saved and time-stamped, helping for reproducibility and book-keeping.

108 Docker
109 -------
110 
111 If you want to use Docker instead, you can build an image using the Dockerfile
112 available in the repository or pulling it from
113 [DockerHub](https://hub.docker.com/r/uclrits/purify).
114 
115 ```
116 docker build -t purify .
117 ```
118 
119 or
120 
121 ```
122 docker pull uclrits/purify
123 ```
124 
125 Then to use it, you should mount the directory with your data and config files
126 to `/mydata` in the container. To run the container and mount the directory is
127 with:
128 
129 ```
130 docker run -it --name purify -v /full/path/to/data:/mydata uclrits/purify
131 ```
132 
133 That will start a shell inside the container in the `/mydata` directory where
134 you can see all the files from your `/full/path/to/data`. There you can run
135 `purify` as shown above.`

160 CMake Tips
161 ----------
162 
163 It is possible to tell ``CMake`` exactly which libraries to compile and link against. The general
164 idea is to add ``-DVARIABLE=something`` to the command-line arguments of CMake. CMake can be called
165 any number of times: previous settings will not be overwritten unless specifically
166 requested. Some of the more common options are the following:
167 
168 - ``CMAKE_PREFIX_PATH``: CMake will look in "CMAKE_PREFIX_PATH/lib"
169  for libraries, "CMAKE_PREFIX_PATH/include" for headers, etc.
170 - ``FFTW3_LIBRARIES``, ``FFTW3_INCLUDE_DIR``
171 - ``BLAS_INCLUDE_DIRS``, ``BLAS_LIBRARIES``
172 
173 All these variables and more can be found and modified in the ``CMakeCache.txt`` file in the build
174 directory. There are extra CMake options sepcific to purify. ``-Ddompi=ON`` will turn MPI on in the build, ``-Dopenmp=ON`` will turn openmp on for the build. ``-Dtests=ON`` will make sure tests are built.
175 
176 ``ctest`` should be run to make sure the unit tests pass.
